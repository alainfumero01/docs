# ONYX FinOps

This folder tracks vendor costs, baseline subscriptions, and usage-based pricing notes for Project Onyx.

Contents:

- `cost-register.md`

## Scope

The cost register is intended to answer three questions:

1. What is already turned on right now?
2. What is the likely baseline monthly cost if current environments remain active?
3. What future or optional costs should we watch before go-live?

## Update Practice

- verify pricing against official vendor pages before updating
- note the verification date in the register
- separate current active costs from optional or future costs
- mark estimates clearly when the exact billed plan is not exposed by an API
